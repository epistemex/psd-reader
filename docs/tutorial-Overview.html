Overview

Revision 2

The psd-reader is simple to use if you need quick access to the bitmap
data of a PSD file.

Simply create a new instance, pass in the URL or an already loaded
ArrayBuffer, and you have an RGBA version that can be converted to canvas:

var psd = new PsdReader({url: "example.psd", onload: function(e) {
    // we now have access to a RGBA version of the file regardless of original format:
    var rgba = this.rgba;
})

If we don't intend to deal with the raw bitmap, we can simply convert it
to a canvas and use for example a 2D context to add graphics, text etc.:

var psd = new PsdReader({url: "example.psd", onload: function(e) {

    var canvas = this.toCanvas();

    // append canvas to DOM
    document.body.appendChild(canvas);
})

We can even scale the bitmap first so the canvas is in the right size
before we use it:

var psd = new PsdReader({url: "example.psd", onload: function(e) {

    // to always get a canvas 800 pixels in width:
    var scale = 800 / this.info.width;
    var canvas = this.toCanvas({scale: scale});
    //...

})

Active vs. Passive modes

Normally, active mode is used. This will do the file loading and conversion.
When done the onload handler will be called.

Sometimes you might want to just load the resource and convert the data
as some later point.

For this, we can use passive mode. Important to note is that in order to
know when data can be converted, which we need to do manually, you have
to provide a callback for the onready event.

To parse the data you would call the parse() method on the instance. This
call is safe to call several times, although discouraged, as it will check
the internal status of the data and only parse if not previously parsed.

var psd = new PsdReader({url: "example.psd", passive: true, onready: onready, onload: onload});

function onready(e) {
    // here we can flag our program ready to parse the file.
    this.parse();    // when done, onload will be called as normal.
})

function onload(e) {
    // here we can access access rgba, or convert to canvas
    var canvas = this.toCanvas();
})

The example above is of course somewhat contrived and unnecessary, but in
a real world example you would for example keep track of a file batch,
count them in the onready handler, when all files had loaded then
parse them. This will allow you to do background loading and eventually
provide a parse mechanism to the user or automatically.

Advanced usage

But we can do a lot more before we create a canvas. We can process the
RGBA data, or file data, directly if we want to.

Chunks

PSD files are as many other graphics (and sound) files chunked, although
in a little bit different way than a typical chunked file such as PNG.
This is often referred to as tagged image format.

PSD files always have 5 chunks, header, color data, resource data, layers
and mask, and finally the image data at the end. Even if there is no data
in a chunk, it will always be present contrary to other graphic formats.

There are disadvantages to this, but the advantage for us is that you
will always find all the chunks in the file which allows us to address
them with a fixed index.

In psd-reader the chunk information can be found in the info object:

var chunks = psd.info.chunks;

and we can access each as an object as well:

var header = chunks[0];
var colorInfo = chunks[1];
var resources = chunks[2];
var layers = chunks[3];
var image = chunks[4];

Each object has a pos and length property which can be used to create
a typed array for parsing:

if (colorInfo.length) {
    var buffer = new DataView(psd.buffer, colorInfo.pos, colorInfo.length)
}

We can now parse the chunk for the information we need. If the chunk is
empty the length will be 0.

Note that all data found through these objects are raw and compressed
data (if originally compressed).

For the resources chunk, we do provide a method to locate a resource with
a certain ID. For example, to locate a thumbnail resource you could do:

var thumb = psd.findResource(1036);  // ID 1036 = thumbnail PS ver > 4
if (thumb) {
    var buffer = new DataView(psd.buffer, thumb.pos, thumb.length);
    // process the data here...
}

You will have to manually process the data though.

Channels and bitmaps

To access the parsed channel bitmaps, which are always uncompressed but
not processed, you can use the bitmaps property of the info object:

var bitmaps = psd.bitmaps;

Use the channels property to know how many bitmaps there are (or simply
use the length property on the bitmaps property):

var channels = psd.info.channels;
var channels = psd.bitmaps.length;

Each channel in a PSD file contain all data for it, there is no interleaving.

To build a single pixel you would have to pick a byte (or word, float etc.)
from each channel and interleave manually, for example, if we have 8-bit RGBA
data in our channels we will have four channels. To create the first pixel
we need to do:

 var dstBuffer = new Uint8Array(someLength);
 dstBuffer[0] = bitmaps[0][0];        // first byte in red channel
 dstBuffer[1] = bitmaps[1][0];        // first byte in green channel
 dstBuffer[2] = bitmaps[2][0];        // first byte in blue channel
 dstBuffer[3] = bitmaps[3][0];        // first byte in alpha channel

Note that with formats such as indexed, bitmap and so forth, you need to
process and convert the original formats manually. In addition with formats
such as L*ab you will also have to do color space conversion.

If you don't need the RGBA data you can turn off the conversion when
creating the PsdReader instance:

var psd = new PsdReader({url: "example.psd", noRGBA: true, onload: function(e) {

    // here we can access the header and channel data - we do not have RGBA here
    var channelArray = this.info.channels;
    var width = this.info.width;
    var height = this.info.height;
    var depth = this.info.depth;
    var byteWidth = this.info.byteWidth;
    var colorMode = this.info.colorMode;  // see also colorDesc

    // if we want to manually convert to RGBA we can also do that here:
    this.toRGBA(callback);
})

The image data

Unless the noRGBA option had been used, and the file is not corrupted or
other errors occurred, the rgba property on the instance will always
contain a processed bitmap in 8-bit RGBA format (ie. 4 bytes per pixel
just as with Canvas).

This bitmap can be useful if you don't need to add any graphics or text
to the image but just need it for processing purposes.

Process it as you would with data from a canvas through the getImageData()
method. You can also set the data directly to a canvas ImageData object -
just make sure they are of the same width and height:

var idata = ctx.createImageData(psd.info.width, psd.info.height);
idata.data.set(psd.rgba);
ctx.putImageData(idata, 0, 0);

The toCanvas() method can do this for you as well as scaling the image
using direct scaling, or a high-quality mode when down-sampling the image
from a large size to a much smaller size (< 50% of original):

var canvas = psd.toCanvas({scale: 0.25, hq: true});

This is a relative slow process so be careful with really large images
(at the time of writing this HQ down-sampling is not yet async and block
based, only loading and decoding - you can of course do this manually or
use other techniques).
