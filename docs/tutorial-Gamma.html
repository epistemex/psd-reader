Gamma

psd-reader can optionally gamma-correct the images for the screen
during the loading and decoding process.

It comes with two separate settings, one for 32-bit color mode, and one
for all others.

By default the gamma is 1, ie. no processing, for all formats, except
for 32-bit mode which by default try to detect the system and applies
the detected gamma automatically. This can of course be overridden.

As we don't have access to the host system from JavaScript we have to
make a guess. If Mac is detected the guessed gamma for 32-bit mode is
1.8, otherwise 2.2 (one for Linux may be included if it turns out to be
necessary). You can always supply static values through the options.

Gamma is defined by options when the psd-reader instance is created.
The gamma is given as inverse value (ie. 1/gamma):

var psd = new PsdReader({
  ...
  gamma  : 1/2.2,    // defaults to 1
  gamma32: 1/2.2,    // defaults to system detected gamma
  ...
  });

Set both to 1 to disable gamma correction.

PSD files appear to be saved with no gamma when in 32-bit mode. This is
likely due to 32-bit being floating point and has a high dynamic range
and is intended for HDR use, although there is no official documentation
for why this is.

Note that applying gamma will alter the output values for RGBA format only.
Gamma is not applied on the internal original format which also can be
accessed.
